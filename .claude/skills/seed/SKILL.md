## Project Seeder Conventions

**Namespace & location:**
```
database/seeders/<EntityName>Seeder.php
```

**Class structure:**
```php
<?php

namespace Database\Seeders;

use App\Models\SomeModel;
use Illuminate\Database\Seeder;

class SomeModelSeeder extends Seeder
{
    public function run(): void
    {
        // seed logic here
    }
}
```

**Key rules:**
1. **Never put data in migrations** вЂ” always in seeders. Migrations are structural only.
2. Use `firstOrCreate(['unique_key' => value], [...other fields])` to make seeders idempotent (safe to run multiple times).
3. ULID PKs are auto-generated by `HasUlids` вЂ” never manually set `id`.
4. Monetary amounts are stored as strings with 4 decimal places: `'1500.0000'`.
5. If the seeder depends on other seeders (e.g., Products need Units and Categories), resolve parent records by a stable unique field (`name`, `slug`, `code`) using `->firstOrFail()` вЂ” never hardcode IDs.
6. Register in `DatabaseSeeder.php` if it should run by default, or leave standalone for manual use with `php artisan db:seed --class=`.

**Ordering dependencies:**
Check if the entity depends on:
- `Unit` (for Products) вЂ” resolve by `name`
- `ProductCategory` вЂ” resolve by `slug`
- `Warehouse` вЂ” resolve by `name`
- `Treasury` вЂ” resolve by `name`
- `Customer` / `Supplier` / `Shareholder` вЂ” resolve by `name` or `phone`
- `Account` (GL) вЂ” resolve by `code`

**Example pattern вЂ” transactional data (invoices, payments):**
Use factories for test/demo data, not hardcoded arrays:
```php
use App\Models\SalesInvoice;
use App\Models\Customer;

public function run(): void
{
    $customers = Customer::all();

    foreach ($customers as $customer) {
        SalesInvoice::factory()->count(3)->create([
            'customer_id' => $customer->id,
            'status'      => 'posted',
        ]);
    }
}
```
